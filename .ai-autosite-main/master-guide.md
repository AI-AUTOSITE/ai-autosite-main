## 📌 絶対原則

### 3つのZERO

🚫 **Zero Ads** - 広告は一切表示しない  
🚫 **Zero Tracking** - Google Analyticsすら使わない  
🚫 **Zero Sign-up** - メールアドレスすら要求しない

### なぜこの原則を貫くのか

**ユーザーの時間は貴重**

- 広告で時間を奪わない
- クッキー同意で操作を中断させない
- 会員登録で使用開始を遅らせない

**プライバシーは基本的人権**

- トラッキングは監視と同じ
- データ収集は信頼の裏切り
- 個人情報要求は過剰な権力

**シンプルは最高のUX**

- 見た瞬間に使える
- 説明不要のインターフェース
- 余計な機能は削ぎ落とす

## 🛠️ 技術スタック（固定）

### 採用技術

✅ **Next.js 14** (App Router)  
✅ **React 18**  
✅ **TypeScript**  
✅ **Tailwind CSS v3**  
✅ **Vercel** (デプロイ)  
✅ **GitHub** (バージョン管理)

### 禁止事項

❌ **Next.js 15** - 最新版は避ける  
❌ **Turbopack** - 実験的機能  
❌ **Tailwind CSS v4** - まだ安定していない  
❌ **React 19** - 安定版を待つ  
❌ **実験的機能全般** - リスクを取らない

### なぜ安定版にこだわるのか

- **予測可能性**: 既知の問題のみに対処
- **サポート**: コミュニティの知見が豊富
- **保守性**: 長期的なメンテナンスが容易
- **信頼性**: ユーザーに迷惑をかけない

## 🎯 開発の黄金律

### UI設計原則

**説明不要 = 見た瞬間に使い方が分かる**

- ラベルやヘルプが最小限
- 直感的な配置とアイコン
- 一般的なUIパターンに従う

**3秒ルール = 3秒以内に操作開始できる**

- ページ読み込み: 1秒以内
- 理解: 1秒以内
- 操作開始: 1秒以内

**Simple English = 小学生レベルの英語のみ**

- 専門用語を避ける
- 短い文章（15語以内）
- 能動態を使う

**Local First = 可能な限りブラウザで完結**

- サーバー通信を最小化
- オフラインでも動作
- プライバシーを守る

### コーディング原則

**動作優先**

- まず動くものを作る
- 最適化は後回し
- 完璧を目指さない

**段階的実装**

- 小さく始める
- 一つずつテスト
- 徐々に機能追加

**可読性重視**

- 変数名は明確に
- コメントは最小限
- ファイルは分割

## 🎨 デザイン哲学

### ミニマリズム

**余白を恐れない**

- 詰め込まない
- 呼吸できる空間
- 目的に集中

**色は控えめに**

- 白黒グレーが基本
- アクセントカラーは1-2色
- グラデーションは慎重に

**フォントは読みやすく**

- 本文: 16px以上
- 行間: 1.5-1.8
- コントラスト比: 4.5:1以上

### アクセシビリティ

**すべての人が使える**

- キーボードナビゲーション
- スクリーンリーダー対応
- カラーブラインド配慮

**WCAG 2.1 AA準拠**

- コントラスト比の確保
- フォーカスの可視化
- エラーメッセージの明確化

## 🔒 プライバシーポリシー

### データ収集の原則

**収集しない**

- 個人情報
- 閲覧履歴
- 使用統計
- Cookie（必須Cookie以外）

**ローカル処理**

- ブラウザで完結
- サーバーに送信しない
- 一時的なメモリ使用のみ

**透明性**

- プライバシーポリシーで明記
- ツールページで表示
- 疑問があれば即答
